HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element appears in the browser's title bar
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
  • An element refers to the entire element from opening tag to closing tag
  • A tag refers only to what appears inside the angle brackets
  • h1,h2,h3 etc...are elements representing the level of heading a given text block represents
  • it's important to keep h1, h2 etc in order, and h1,the largest header, should only appear once
  • p element represents a paragraph or block of text.
  • ul is for unordered lists
  • ol is ordered lists
  • li is list items
  • img contains information about images displayed on the page
  • src defines location of the image
  • br creates a line of empty space
  • a stands for anchor; creates links to the same webpage or other webpages
  • href points to the URL for the link
  • header appears at top of page and generally contains a navigation menu and/or introductory content
  • The `alt` attribute contains a text string that describes the appearance and functionality of the image
  • The footer appears at the bottom of the page and contains the author, copyright, contact, sitemap, and navigation

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • the default branch is main
  • a feature branch is where we make commits of all the changes we make to the code
  • a pull request (PR), is used to merge a comparison branch into the base branch
  • git add -A; add commandadds modifications in current working branch to the staging area, and -A flag indicates we want to add all changes
  • git commit -m "add 'enter what the commmit contains"
  • git identifies a git command
  • pull receives a branch's modification into the local environment
  • origin indicates the source of the pull will be in the GitHub repo
  • push command pushes changes we have locally to our remote GitHub branch
  • use '-m' flag to associate a message with our commit
